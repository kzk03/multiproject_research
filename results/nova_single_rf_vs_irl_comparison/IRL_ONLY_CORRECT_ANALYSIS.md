## エグゼクティブサマリー

IRLのみ正解した開発者を分析した結果、**IRLは低受諾率の離脱者を正確に検出する能力**に優れていることが判明しました。

### 主要な発見

1. **離脱者の検出が得意**
   - 離脱者: 11件（68.8%）
   - 継続者: 5件（31.2%）
   - IRLは特に**低受諾率（平均2.9%）の離脱者**を正確に予測

2. **RFの過信を防ぐ**
   - RFは受諾率が低くても、履歴リクエスト数が多いと「継続」と誤判定
   - IRLは受諾率の低さを正しく評価し、「離脱」と予測

3. **ボーダーラインケースの正確な判定**
   - IRL平均確率: 0.461（閾値0.471に-0.010）
   - RF平均確率: 0.496（閾値0.5に-0.004）
   - **非常に微妙な判定**をIRLが正しく行っている

## IRLの判定メカニズム推測

### 1. 受諾率重視の判定

**離脱判定**:
- 受諾率 < 10% → 高確率で離脱
- 受諾率0-3%の範囲が特に確実

**継続判定**:
- 受諾率 > 20% → 高確率で継続
- リクエスト数が少なくても受諾率を優先

### 2. リクエスト数の影響が限定的

- RFと違い、リクエスト数1724でも受諾率0.3%なら「離脱」
- リクエスト数3でも受諾率20%なら「継続」の可能性

### 3. ボーダーライン判定の精度

**IRL確率の分布**:
- 最小: 0.438
- 最大: 0.482
- 範囲: 0.044（非常に狭い）
- 閾値0.471付近に集中

**洞察**: IRLは確信度が低いケースでも、微妙な差で正確な判定。

## 代表的な事例

### 事例1: nova_hyperv_ci@cloudbasesolutions.com

**最も顕著なRFの過信事例**

```
真のラベル: 離脱 (0)
履歴: 1724 リクエスト（全開発者中最多）
受諾率: 0.3%（極めて低い）

IRL: 0.452 → 離脱 ✓ 正解
RF:  0.630 → 継続 ✗ 不正解（大幅な過信）

期間: 0-3m → 0-3m, 0-3m → 3-6m（2回出現）
```

**IRLの判断**:
- 1724という圧倒的なリクエスト数にも関わらず
- 受諾率0.3%という致命的な低さを正しく評価
- → 離脱と予測

**RFの誤り**:
- リクエスト数1724を過大評価
- 受諾率0.3%を無視
- → 継続と誤判定（RF確率0.63）

**教訓**: **量よりも質**。活動量が多くても、受諾率が極端に低ければ離脱する。

---

### 事例2: kchamart@redhat.com

**最も顕著なRFの過小評価事例**

```
真のラベル: 継続 (1)
履歴: 46 リクエスト
受諾率: 34.8%（全開発者中最高）

IRL: 0.475 → 継続 ✓ 正解
RF:  0.250 → 離脱 ✗ 不正解（大幅な過小評価）

期間: 9-12m → 9-12m
```

**IRLの判断**:
- 受諾率34.8%という高品質を正しく評価
- リクエスト数46は十分な活動量
- → 継続と予測

**RFの誤り**:
- リクエスト数46を過小評価
- 受諾率34.8%を評価できず
- → 離脱と誤判定（RF確率0.25、極端に低い）

**教訓**: **質の高い貢献者を見逃すな**。受諾率30%超は継続の強いシグナル。

### 事例3: christian.rohmann@inovex.de

**最も驚異的なIRLの予測事例**

```
真のラベル: 継続 (1)
```

**RFの限界**:
- 受諾率0%・リクエスト3という表面的データのみで判断
- → 離脱と誤判定（RF確率0.08、最低値）

**教訓**: **IRLの時間的パターン学習能力は、表面的データを超える**。
